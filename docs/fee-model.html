Methodology & Assumptions
Headline accuracy: When the inputs match the live product defaults ($2,400 start + $200/mo, 0.75% platform, 10% performance), the summary's Ending NAV and platform/performance/exchange figures are read straight from the simulator (ef_run_lth_pvr_simulator, USDPC 10% APY on) and match the public back-tester / Admin simulator exactly (✓). Change any rate or contribution and the model falls back to the monthly return-extraction estimate described next.
Portfolio model (estimate): Monthly returns extracted from the LTH PVR back-test. Pure return isolated: r = (nav_next − monthly_contrib) / nav_prev − 1. Applied to a fresh $2,400 start with $200/month additions. Sub-periods (1yr, 3yr, 7yr) use the most-recent N months, rebased to a fresh start.
Management fee: charged monthly = NAV × (rate ÷ 12) on end-of-month portfolio value. (Hypothetical alternative structure — not part of the live product, so it is never anchored.)
Exchange fee sharing: 50% of VALR fees using simulator rates: USDT/ZAR 18 bps maker (contribution conversion); BTC/USDT 8 bps maker (trade fee, same as maker_bps_trade=8 in back-tester); USDPC/USDT 10 bps taker (market order, matches USDPC_DEFAULT_TAKER_FEE_RATE). For the anchored product config the rebate is the simulator's actual per-day total across all three legs — every BUY converts USDPC→USDT then USDT→BTC, every SELL the converse, so the rebate varies with the real daily order size and HOLD days earn nothing. Each period's daily rebates sum exactly to the anchored period total. Custom rates/ranges fall back to a per-contribution estimate (amt × ½ × rate per leg).
Performance fee: 10% of monthly profit above the high-water mark (HWM).
Drill-down: Raw LTH PVR back-test NAV for the original simulation portfolio (daily when chart-data-daily.js is present, otherwise weekly). At the daily level, fees are attributed to the day they are charged: platform on the contribution day (month start), exchange rebate on each BUY/SELL trade day (real daily order sizes, anchored config), management & performance at month end. Monthly return formula: r = (first_day_next_month − contrib) ÷ first_day_this_month − 1.
Date range: End Date re-anchors the 1/3/5/7/10-year windows to end on the chosen date (computed backwards from there). Setting both Custom Range dates replaces the standard periods with a single window covering exactly that span. Custom dates use the reconstruction estimate (the ✓ anchors apply only to the standard periods ending on the latest data date).
Export: Export Excel downloads a four-tab .xlsx workbook — Summary (per-period totals), Yearly Data, Monthly Data and Daily Data (every period's daily back-test NAV, daily return and attributed fees). Only the fee columns whose toggles are ON are included.
