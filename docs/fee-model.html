Methodology & Assumptions
Headline accuracy: When the inputs match the live product defaults ($2,400 start + $200/mo, 0.75% platform, 10% performance), the summary's Ending NAV and platform/performance/exchange figures are read straight from the simulator (ef_run_lth_pvr_simulator, USDPC 10% APY on) and match the public back-tester / Admin simulator exactly (✓). Change any rate or contribution and the model falls back to the monthly return-extraction estimate described next.
Portfolio model (estimate): Monthly returns extracted from the LTH PVR back-test. Pure return isolated: r = (nav_next − monthly_contrib) / nav_prev − 1. Applied to a fresh $2,400 start with $200/month additions. Sub-periods (1yr, 3yr, 7yr) use the most-recent N months, rebased to a fresh start.
Management fee: charged monthly = NAV × (rate ÷ 12) on end-of-month portfolio value. (Hypothetical alternative structure — not part of the live product, so it is never anchored.)
Exchange fee sharing: 50% of VALR fees using simulator rates: USDT/ZAR 18 bps maker (contribution conversion); BTC/USDT 8 bps maker (trade fee, same as maker_bps_trade=8 in back-tester); USDPC/USDT 10 bps taker (market order, matches USDPC_DEFAULT_TAKER_FEE_RATE).
Performance fee: 10% of monthly profit above the high-water mark (HWM).
Drill-down: Raw LTH PVR back-test NAV for the original simulation portfolio (daily when chart-data-daily.js is present, otherwise weekly). Monthly return formula: r = (first_day_next_month − contrib) ÷ first_day_this_month − 1.
Disclaimer: Back-test data June 2016–June 2026. Past performance does not guarantee future results.
